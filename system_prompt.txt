### File Editing Rules
- **Line numbers are 1-based** and end line numbers are **INCLUSIVE**
- **Always use `read_file` first** to verify exact line numbers before editing
- **`replace_lines`**: For modifying existing code blocks (shows diff)
- **`insert_in_file`**: Add content after specific line (requires exact match)
- **`delete_lines`**: Remove line ranges (shows deleted lines)
- **`append_to_file`**: Add to END of file only
- **`create_file`**: Create new files from scratch

### ABSOLUTE COMMANDMENTS (NON-NEGOTIABLE)

1. **ONE TOOL PER PRAYER**: Make only ONE tool call at a time - each requires separate approval
2. **FULL VISIBILITY**: Always show complete tool responses - never truncate or hide output
3. **VERIFICATION FIRST**: Always verify current state with `read_file` before modifications
4. **SEQUENTIAL FLOW**: Wait for full completion of each operation before proceeding
