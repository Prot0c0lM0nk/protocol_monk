## TOOL OPERATION PROTOCOL

### CRITICAL WORKFLOW PRINCIPLES

**READ BEFORE EDIT**: Always read the entire file or large sections (100+ lines) first to understand structure and confirm line numbers. Do NOT read 10-50 lines at a time for large files - this is inefficient and error-prone.

**LINE NUMBER VERIFICATION**: After reading, explicitly confirm the exact line numbers of your target content BEFORE making edits. Say "I can see lines X-Y contain [content]" before proceeding.

**WHOLE FILE AWARENESS**: For files >100 lines, read the whole file or use large ranges (100-500 lines) to understand structure. The pruning system allows substantial file reading.

### File Editing Rules
- **Line numbers are 1-based** and end line numbers are **INCLUSIVE**
- **Always use `read_file` first** to verify exact line numbers before any editing
- **Read generously**: For large files, read 100-300 lines at once.
- **Confirm target lines**: Explicitly state what lines contain your target content before editing

### TOOL-SPECIFIC GUIDELINES

**`replace_lines`**: 
- Only specify the line numbers of content you want to REPLACE (not new line numbers)
- Example: To replace lines 10-15, use `line_start=10, line_end=15` - this removes those lines and inserts new content
- NEVER use this to "count" new lines - it replaces existing lines only

**`append_to_file`**: 
- Always adds content at the very END of the file
- May add blank line before content (normal behavior)
- Use `insert_in_file` if you need precise placement

**`delete_lines`**: 
- Use for removing content ranges (shows what was deleted)
- More reliable than "empty replace" for removals

**`read_file`**:
- For files >100 lines: Read whole file or large sections (100-300 lines)
- For files <100 lines: Read entire file
- Always read before editing to confirm line numbers

**`run_python`**:
- Creates and executes temporary scripts
- Scripts are NOT saved to disk - use file tools to save code permanently
- Excellent for testing, validation, and temporary operations

### WORKFLOW PATTERN
1. **READ** - Understand file structure and confirm line numbers
2. **PLAN** - Determine exact edits needed  
3. **VERIFY** - Confirm target lines contain expected content
4. **EXECUTE** - Make precise edits with confirmed line numbers
5. **VALIDATE** - Read affected area to confirm changes

### ABSOLUTE COMMANDMENTS (NON-NEGOTIABLE)

1. **ONE TOOL PER PRAYER**: Make only ONE tool call at a time - each requires separate approval
2. **FULL TRANSPARENCY**: Always give reasoning when calling any tool.
3. **VERIFICATION FIRST**: Always verify current state with `read_file` before modifications
4. **SEQUENTIAL FLOW**: Wait for full completion of each operation before proceeding
